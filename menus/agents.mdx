Agents are the core of Beni. Each agent is an AI assistant you configure to handle customer conversations across chat, voice, and SMS.

## Agent List

The Agents page shows all agents in your workspace. Each agent card displays:

- **Name** and description
- **Status** — `Active` or inactive
- **Last Active** — when the agent last handled a conversation

Use the **+ New Agent** button in the top right to create a new agent. If you have no agents yet, you'll see an empty state prompting you to create your first one.

---

## Creating an Agent

Click **+ New Agent** to open the multi-step agent creation wizard. Move through each step using the **Previous** and **Next** buttons, or jump to any step using the sidebar.

### Step 1 — Basic Info

Set up the core identity of your agent.

| Field | Description |
|-------|-------------|
| **Agent Name** | Visible to your team. e.g. `Website AI Assistant`, `Customer Support Bot` |
| **Description** | A brief summary of what this agent does |
| **Language** | The language the agent will communicate in (default: English) |
| **Active Status** | Toggle to enable or disable the agent |

### Step 2 — Prompts

Define how your agent communicates using a **System Prompt** — this sets the agent's personality, role, and behavioral guidelines.

A default prompt is provided to get you started. Refer to the **Best practices guide** link next to the System Prompt label for tips on writing effective prompts.

<Tip>Keep system prompt instructions concise for better conversations.</Tip>

### Step 3 — Intelligence

Choose the LLM configuration that powers your agent.

- **Automatic Model Selection** — Beni automatically selects the best AI model for your agent to ensure optimal performance and cost efficiency
- **Temperature** — Controls response style on a scale from `More precise` (0) to `More creative` (1.0)

<Info>Lower values (0–0.3) make responses more focused and deterministic. Higher values (0.7–1.0) make responses more diverse and creative. For conversational agents, we recommend 0.3–0.5 for consistency.</Info>

### Step 4 — Knowledge Base

Select the documents your agent can reference when answering questions. Search or browse files and folders from your Knowledge Base. Click **Manage** to add or manage documents.

### Step 5 — Tools & Integrations

Give your agent the ability to perform actions and access external systems. Only active tools and connected integrations are shown. Select the tools you want to enable for this agent, or click **Manage** to create and configure new tools.

<Tip>You can filter by tool type using the **All Types** dropdown to quickly find API tools or integrations.</Tip>

### Step 6 — Escalation

Configure where to notify when a user requests a human agent.

**Use global notification settings** — when enabled, this agent inherits the organization-wide escalation settings configured in **Settings → Notifications**. No further configuration is needed on the agent.

When disabled, you can configure agent-specific escalation settings:

| Option | Description |
|--------|-------------|
| **Enable email notifications** | Send an email alert when this agent escalates to a human |
| **Recipient Emails** | Add up to 10 email addresses to notify on escalation. Use **+ Add email** to add more recipients |

<Info>To configure organization-wide escalation defaults, go to **Settings → Notifications**.</Info>

### Step 7 — Chat Widget

Customize the chat widget that appears on your website.

| Field | Description |
|-------|-------------|
| **Display Name** | Name shown in the chat widget header. Leave empty to use the agent name |
| **Primary Color** | Brand color for the widget (hex code) |
| **Greeting Message** | Greeting shown when the chat widget opens. Leave empty to use the agent's default first message |
| **Bubble Text** | Text shown on the chat bubble before the user opens it |

A live **Preview** (380×520 px) updates as you make changes.

Click **Create Agent** in the top right to finish and save your agent.

## Updating an Agent

To edit an existing agent, click on it from the Agents list. The page opens as **Configure Agent** with all the same steps as creation. All settings are identical — including the Escalation step — except for one addition in the Chat Widget step.

When you're done making changes, click **Save Changes** in the top right to save. You can also click **Preview Widget** to see how your chat widget looks before saving.

### Chat Widget — Embed Code

When editing an existing agent, an **Embed Code** section appears below the Chat Widget Appearance card. Add this script to your website to display the chat widget.

```html
<script
  src="https://widget.trybeni.ai/widget/latest/beni.iife.js"
  data-agent-id="YOUR_AGENT_ID"
  data-publishable-key="YOUR_PUBLISHABLE_KEY"
  defer
></script>
```

Place this script tag just before the closing `</body>` tag of your HTML.

<Info>Replace `YOUR_PUBLISHABLE_KEY` with a publishable key from **Settings → Developer → Publishable Keys**. Publishable keys are safe to use in frontend code and can be restricted to specific domains.</Info>

<Tip>See the [installation guide](/chat-widget/installation) for detailed setup instructions.</Tip>
